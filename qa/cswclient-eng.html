Searching in catalogs with QGIS and CSWclient

How to work with catalog services in QGIS

CSW (Catalogue Service for Web) is an OGC (Open Geospatial Consortium) specification, that defines common interfaces to discover, browse, and query metadata about data, services, and other potential resources.

CSWclient plugin for QGIS provides userfriendly interface for this services.

Development of CSWclient was sponsored by Linifiniti Consulting.

1. Installation notes

Plugin is available via GIS-Lab repository.

CSWClient is designed for QGIS version 1.5.0 and higher. There are no dependencies, all necessary files (owslib) are included in package.

2. How to work

After plugin installation and startup by clicking main window with two tabs will appear.

Server list management

The «Servers» tab is used for managing the list of available servers. Plugin is shipped with small default list of servers, which you can add by pressing «Add default servers» button. You can also add new servers, edit or delete existing ones with «New», «Edit» and «Delete» buttons respectively.

Besides this it is possible to share your list of servers with others by saving server list in file with «Save» button and load them from file with «Load» button. Before saving or loading you can select whichever servers you'd like to share. This is useful when you need to share settings between computers and people.

«Server info» will display available information about server in the text area. Button «Metadata» has same purpose, but displayed information in separate window as XML document.

Searching for data

«Search» tab has everything you need to set up search parameters and view results.

First of all you need to enter search keyword (or comma separated list of keywords) and geographic boundaries to search in. By default, canvas bounding box is used. If necessary, you can enter values in fields manually. «From map canvas» and «Set global» buttons allow to set up canvas bbox and global bbox in one click.

Results are displayed in list. It is possible to sort them by clicking on column title. When record is selected «Abstract» field is filled, and if available dataset URL is displayed. «Metadata» button also becomes active. Pressing on this button will display record metadata. When «Show XML» is clicked a window will appear with server response in XML format.

You can navigate through search results with «First», «Last», «Next» and «Prev» buttons.

Accessing data

If record has type «downloadableData» and there is a link in their metadata, button «Open URL» will be active. Pressing this button will open the link in the default browser.

If record has type «liveData» and there is a link in their metadata, «Add to WMS list» button will be enabled. When this button is pressed plugin will try to verify if this WMS server is valid. On success you will be asked for name for this server and it will be added to the QGIS WMS servers list. You can connect for this server as usual using Layer\Add WMS layer.

Note. Current test for WMS URL's validity is very simple. So in some cases this test will return wrong results and server will not be added to the QGIS WMS list. In this case you can copy URL and add it manually. We'd also welcome report on that.
