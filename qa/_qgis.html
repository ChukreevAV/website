Обзор программного пакета QGIS

Краткая аннотация

Данная статья представляет собой обзор программного пакета QGIS, который является одним из самых динамично развивающихся, свободно распространяемым пакетом для работы с пространственными данными. Благодаря своей функциональности, QGIS может составить неплохую альтернативу распространенным коммерческим пакетам работы с ГИС, таким как Arcview/ArcGIS, Mapinfo и другим. Уникальной особеннойстью QGIS является открытость его исходного кода, что позволяет вносить изменения и легко добавлять нужные новые функции.

Статья основана на Руководстве пользователя QGIS версии 0.9, авторы статьи так же проводят параллели и сравнения с другими распространенным программным обеспечением для ГИС. Более подробную информацию о QGIS можно найти в этом руководстве.

Введение

QGIS - свободно растпространяемый программный пакет с открытым исходным кодом предназначенный для управления, создания и редактирования пространственных данных. QGIS был создан в 2002 году, для его разработки используется C++ и Qt (библиотека элементов интерфейса). QGIS работает в большинстве распространенных операционных систем: Windows, Unix и OS X. Исторически QGIS создавался как программа просмотровщик пространственных данных, но к настоящему моменту он "оброс" большим количеством модулей расширения и его текущая функциональность включает следующие основные возможности:

  • поддержка растровых и векторных данных
  • поддержка таблиц PostreSQL через PostGIS
  • просмотр, редактирование и анализ данных GRASS
  • макетирование карт
  • перепроектирование данных, в том числе "на лету"
  • выборки
  • поддержка OGC и экспорт проектов для Mapserver
  • многое другое...

Установка

Текущая версия QGIS - 0.9, ее можно загрузить по адресу http://download.qgis.org, размер дистрибутива без примеров данных составляет 68 Мб. При установке, есть возможность установить так же два набора данных, которые будут загружены дополнительно. Один из этих наборов представляет собой базу данных GRASS (Spearfish), второй - набор растровых и векторных слоев в формате shape-файлов (Alaska). В этой статье иллюстрации основаны на этих наборах данных.

Начало работы

Интерфейс

После загрузки QGIS появляется основное окно проекта (рис 1), основными частями которого являются меню, панели инструментов, легенда, окно карты, окно обзорной карты, строка статуса.

Легендой карты в QGIS называется аналог содержания (TOC - Table of contents Arcview/ArcGIS), в легенде задается порядок отображения слоев, их видимость и условные обозначения. Слои как и в ArcGIS могут объединяться в группы, группы помимо лучше структурирования содержания, также позволяют быстро менять условные обозначения сразу нескольким веторным слоям.

Обзорная карта позволяет быстро перемещатья по основной карте, добавлять и удалять слои из обзорной карты можно из легенды, обычно в качестве обзорного выбирается один-два основных слоя.

Строка статуса показывает текущие координаты курсора, индикатор прогресса если идет выполнение какого-либо процесса, географический охват карты. Иконки на строке статуса позволяют временно показать параметры текущей системы координат и приостановить перерисовку слоев (render). Остановка перерисовки может быть полезна, например, при загрузке множества слоев и сначала изменении их легенды, а потом отрисовки.

Проекты

Аналогично большинству программных пакетов для работы с ГИС, текущая рабочая сессия может быть сохранена в специальный файл - проект. В проекте сохраняется перечень используемых слоёв и их параметры, включая легенду, проекция карты, текущий географический охват окна с картой. Формат файла проекта - XML, таким образом он может быть отредактирован в любом редакторе. В проекте не сохраняются сами геоданные.

Работа с векторными данными

QGIS позволяет работать с векторными данными в нескольких форматах, включая те, которые могут быть импортированы с помощью OGR. Вот некоторые из них:

  • ESRI Shapefile, *.shp
  • Arcinfo Coverage
  • Mapinfo, *.mif/*.mid, *.tab
  • Spatial Data Transfer Standard, *catd.ddf
  • GML, *.gml

Помимо этих форматов, QGIS также позволяет загружать слои PostGIS хранимые в базе данных PostgreSQL. Для работы с этими данными необходимо создать подключение к базе данных, подключиться и выбрать загружаемый слой из списка. Помимо слоев содержащих геометрические объекты, из PostreSQL можно также загружать таблицы и виды.

Помимо загрузки векторных данных из PostGIS, с помощью специального модуля QGIS SPIT (Shapefile to PostGIS Import Tool) можно также загружать данные из shape-файлов в PostGIS.

Условные обозначения

Как и любая другая ГИС, QGIS позволяет работать с легендой векторных слоев и визуализировать данные тремя типами легенды:

  • Уникальные значения
  • Классификация
  • Непрерывная шкала

Помимо этого, QGIS предоставляет средства работы с символами различных типов, позволяет использовать растровые заливки для векторных объектов, работать с прозрачностью векторных слоев и многое другое. Вызов редактора условных обозначений слоя осуществляется через свойства слоя, закладку условные обозначения.

Метаданные

QGIS также позволяется работать с метаданными векторных слоев. Как и с легендой, доступ к метаданным слоя осуществляется через свойства слоя, закладку метаданные.

Метки

QGIS расставлять метки для объектов слоя, используя одно из его атрибутивных полей в качестве источника.

Действия

Используя закладку действия в свойствах векторного слоя, можно назначать определенные действия в зависимости от содержимого используемого поля атрибутивной таблицы. Эти действия могут помочь например запускать программу браузер по ссылке, содержащейся в поле или производить еще какие-то операции.

Использовать заданное действие можно из открытой атрибутивной таблицы слоя (правой кнопкой мыши на записи) или из окна идентификации.

Создание слоев и их редактирование

Данная версия QGIS позволяет создавать только слои в формате shape. С помощью расширения GRASS можно также создавать данные в формате GRASS.

QGIS предоставляется достаточно широкий набор инструментов для редактирования новых и существующих векторных слоев. С помощью него можно:

  • Создавать объекты (точки, линии, полигоны)
  • Перемещать выбранные объекты
  • Разделять объекты
  • Удалять объекты
  • Добавлять, удалять и перемещать узлы линейных и полигональных объектов
  • Прорезать "дырки" в полигонах и создавать островные полигоны
  • Вырезать, копировать и вставлять объекты

Подобно другим пакетам ГИС, данные в QGIS защищены от случайного редактирования, для начала их изменения необходимо перейти в режим редактирования.

Для более точного редактирования, существует возможность настройки параметра примыкания (snapping) узлов нового объекта к узлам существующих объектов редактируемого слоя.

Работа с растровыми данными

Текущая версия QGIS поддерживает работу практически со всеми наиболее распространенными растровыми форматами:

  • Arc/Info Binary Grid
  • Arc/Info ASCII Grid
  • GRASS Raster
  • GeoTIFF
  • Spatial Data Transfer Standard Grids (с некоторыми ограничениями)
  • USGS ASCII DEM
  • Erdas Imagine

Так как загрузка растровых данных в QGIS осуществляется с помощью GDAL, другие формат, с которыми работает GDAL скорее также можно будет загрузить в QGIS, но тестирования работы с ними пока произведено не было.
