Производительность аппаратного обеспечения в операциях с пространственными данными

Открытое исследование пытающееся ответить на вопрос какие компоненты аппаратного обеспечения больше всего влияют на производительность в операциях с пространственными данными.

Приглашаем принять участие в исследовании производительности различных конфигураций аппаратного обеспечения с целью понять какие из компонент имеют наиболее сильное влияние на производительность и следовательно во что стоит вкладывать финансы. Планируется начать заполнение таблиц данными по результатам тестов приведенных ниже, а после набора необходимой базы - проанализировать результаты и попытаться сделать определенные выводы. Одновременно, информация представленная в таблице может дать пищу к размышлению и без дополнительного анализа, Вы можете просто сравнить результаты со своими.

Иногда, серия тестов проводится на одном и том же оборудовании, при меняющемся программном обеспечении (например при установки обновлений), следствие этого - появление "дублирующихся" описаний аппаратного обеспечения. Чтобы отразить это в тестах, помимо конфигурации оборудования в таблице отражается ссылка на другие тесты произведенные на этом же оборудовании. Например, Athlon 64x2 6000+ 3.1 ггц, 4x1 гб DDR2 PC6400 800 мгц, Server 2003 Enterprise (=B1) означает, что тот же компьютер использовался для теста B, номер 1.

В описание аппаратного обеспечения должно входить:

  • Процессор
  • Оперативная память
  • Жесткий диск
  • Версия операционной системы
  • Версия ПО включая build (для ArcGIS: Help\About Arcmap)

Из памяти должны быть выгружено все программное обеспечение кроме работающего ПО, включая основные резидентные программы (антивирусы и т.д.). Никаких операций в процессе работы производиться не должно. Результат должен подтверждаться копией экрана (print screen) соответственно окон лога ERDAS Imagine (Session Log), менеджера задач ArcGIS или файла генерируемого скриптом буферизации (см. примечание к тесту B).

Мы не ограничиваемся программными продуктами одного производителя, если вы знаете пакет производящий ту же операцию, что описана в тесте, присылайте нам свои результаты, это сделает возможным сравнить не только аппаратное, но и программное обеспечение.

Тест A. Перепроектирование с распаковкой из MrSID

Данные: мозаика Landsat TM 16974х19734 пикселей (скачать тестовый набор данных) источник: Global Land Cover Facility.
Операция: DataPrep\Reproject Images, исходная ситема координат UTM Zone 38/WGS 1984 (задана самим файлом), конечная система координат Geographic\WGS 84, выходной формат: IMG, размер пиксела: 0.000259х0.000259, единицы: Degrees, режим rigorous transformation - включен, переключатель Ignore zeroes in stats - выключен, интерполяция - nearest neighbor (подробнее).

Измерение времени производится по логу Erdas Imagine (Session\Session log...), точность в секундах. Конечным результатом операции должен являться файл размером 2.5 Gb.

Тест B. Буферизация векторных данных

Данные: Объекты гидросети России (119405 объектов) из VMap0 (скачать тестовый набор данных),
Операция: Буферизация при помощи инструмента Buffer из Arctoolbox (Analysis Tools\Proximity\Buffer) на 1 км (все установки по умолчанию, растворение границ отключено). В новый проект загружен только один слой, фрейм данных не спроектирован.

Примечание: Время выполнение данного теста может значительно изменяться, поэтому разовое определение времени затраченного на его осуществления может быть очень не точным. Если вы хотите получить более объективную оценку - воспользуйтесь специальным скриптом, который произведет 100 операций буферизации и запишет результат каждой в текстовый файл, по этим данным можно будет более точно расчитать среднее время выполнения этой операции на вашей конфигурации. В случае если такие данные предоставлены, они показаны в таблице как N min M sec (±K sec), то есть среднее время плюс-минус одно стандартное отклонение.

Сам скрипт как и данные могут находится в любой директории, перед запуском скрипта его необходимо отредактировать, изменив два параметра - указав директорию где находятся данные и полный путь к файлу набора инструментов анализа ArcGIS (*.tbx):

# расположение данных
gp.workspace = "c:/temp/2"
# расположение тулбокса (папка ArcGIS)
gp.AddToolbox("C:/Gis/ArcGIS/ArcToolbox/Toolboxes/Analysis Tools.tbx")

Если скрипт не работает и выдается ошибка: ImportError: No module named arcgisscripting, значит установлена версия Python меньше 2.4 (например 2.1) и необходимо использовать скрипт buffer-benchmark-21.py. Скрипты проверены и работают для ArcGIS 9.0, 9.2.
